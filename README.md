CLI-роллер для REG.RU CloudVPS: создает серверы, проверяет публичный IPv4, сравнивает его с целевыми подсетями и оставляет только подходящие инстансы.

## Что умеет

- Перед запуском показывает интерактивное меню с основными настройками.
- Создает сервер по параметрам из `config.json`.
- Ждет и проверяет, появился ли публичный IPv4.
- Оставляет сервер, если IP попадает в одну из нужных подсетей.
- Удаляет сервер, если IP не подходит.
- Записывает зависшие инстансы в отложенный список и делает по одному повторному чеку перед каждой новой итерацией.
- Пишет лог в `roller.log`.
- Поддерживает уведомления в Telegram и Discord.

## Требования

- Python 3.10+
- Пакет `requests`

## Установка

```bash
pip install requests
```

Для Windows можно запускать через:

```bash
py main.py
```

## Быстрый старт

1. Откройте `config.json`.
2. Укажите рабочий `api_token`.
3. Проверьте `target_subnets`, `server_payload` и `timings`.
4. Запустите скрипт:

```bash
py main.py
```

По умолчанию откроется меню. Если нужен прямой запуск без меню:

```bash
py main.py --no-menu
```

Также можно переопределить цель по количеству серверов:

```bash
py main.py --max 3
```

Или использовать другой конфиг:

```bash
py main.py --config custom_config.json
```

## Меню

При обычном запуске доступны пункты:

- запуск роллера
- просмотр текущих настроек
- изменение количества целей
- изменение таймингов
- изменение параметров сервера
- изменение API-настроек
- изменение подсетей
- сохранение `config.json`

Изменения в меню не применяются к файлу автоматически, пока вы не выберете сохранение.

## Структура конфига

Пример:

```json
{
    "api_token": "REG_RU_TOKEN",
    "api_base_url": "https://api.cloudvps.reg.ru/v1/reglets",
    "server_payload": {
        "backups": false,
        "floating_ip": true,
        "image": "ubuntu-18-04-amd64",
        "name": "vm",
        "region_slug": "openstack-msk1",
        "size": "c2-m2-d10-base"
    },
    "target_subnets": [
        "79.174.91.0/24",
        "79.174.92.0/24"
    ],
    "max_success": 1,
    "timings": {
        "initial_wait": 90,
        "check_interval": 5,
        "stability_checks": 3,
        "delete_wait": 10
    },
    "notifications": {
        "telegram_bot_token": "",
        "telegram_chat_id": "",
        "discord_webhook_url": ""
    }
}
```

### Основные поля

- `api_token`: токен REG.RU API.
- `api_base_url`: endpoint для работы с инстансами.
- `server_payload`: параметры создаваемого сервера.
- `target_subnets`: список подсетей, в которые должен попадать публичный IP.
- `max_success`: сколько подходящих серверов нужно найти.
- `timings.initial_wait`: сколько ждать после создания сервера до начала проверок.
- `timings.check_interval`: интервал между проверками.
- `timings.stability_checks`: сколько успешных проверок подряд считать стабильностью.
- `timings.delete_wait`: пауза после удаления неподходящего сервера.
- `notifications`: опциональные уведомления об успехе.

## Как работает роллер

1. Создает сервер через API REG.RU.
2. Ждет `initial_wait` секунд.
3. Проверяет состояние сервера и пытается получить публичный IPv4.
4. Если сервер стабилен и IP найден:
   - если IP подходит по подсетям, сервер остается;
   - если IP не подходит, сервер удаляется.
5. Если сервер не стабилизировался или IP пока не появился:
   - инстанс не удаляется сразу;
   - он попадает в список отложенных;
   - перед каждой новой итерацией роллер делает один повторный чек каждого такого инстанса.

### Поведение отложенных инстансов

- Если отложенный сервер позже получил подходящий IP, он засчитывается как найденный.
- Если отложенный сервер позже получил IP, но он не подходит, сервер удаляется.
- Если IP по-прежнему нет, сервер остается в списке до следующей быстрой проверки.

## Остановка

При `Ctrl+C` скрипт не удаляет активный инстанс автоматически. Он спрашивает:

- удалить текущий инстанс;
- или оставить его без удаления.

Отложенные инстансы при этом не трогаются автоматически.

## Логи

Во время работы создается файл:

- `roller.log`

Туда пишутся:

- создание серверов
- ошибки API
- результаты проверок
- удаление инстансов
- события по отложенным серверам

## Частые проблемы

### 401 Unauthorized

Проверьте:

- что в `config.json` указан корректный `api_token`
- что токен не просрочен
- что у токена есть доступ к CloudVPS API

### IP всегда `(еще нет)`

Скрипт пытается извлекать IPv4 из нескольких форматов ответа API. Если IP долго не появляется, сервер уходит в отложенные проверки и будет перепроверяться перед каждой новой итерацией.

### Серверов стало слишком много

Это возможно, если много инстансов попали в отложенный список и еще не были удалены. Следите за логом и итоговой статистикой.
